# WordPunk — CLAUDE.md

## Проект

PWA-приложение для изучения английских слов в стиле киберпанка.
React + TypeScript + Vite + IndexedDB (idb). Без бэкенда — всё локально.

GitHub: https://github.com/emorozoff/wordpunk
Deploy: https://emorozoff.github.io/WordPunk/

## UX-принципы

- **Зона большого пальца**: все интерактивные элементы (кнопки ответов, кнопка ДАЛЕЕ) должны быть внизу экрана, в зоне досягаемости большого пальца на телефоне. Карточка со словом — сверху/в центре, кнопки — снизу.

- **Sentence-first обучение (навсегда)**: все карточки показывают предложение как главный элемент, изучаемое слово выделено внутри предложения. Пользователь учит слова в контексте, не в вакууме. Для многозначных слов (lose = потерять/проиграть) делаем отдельные карточки с разными примерами — каждая раскрывает одно конкретное значение. Единый UX для всех карточек, без исключений.

## База слов

- **Целевой размер базы: 7000 слов** (с запасом — пользователь планирует вручную удалить много слов позже, поэтому нужен буфер).
- Одно английское слово может быть в нескольких карточках, если имеет разные значения (каждое значение — отдельная карточка с примером, раскрывающим именно это значение).
- Счётчик уникальных слов в шапке считается по английскому слову, а не по количеству карточек.

### Удаление карточек — ВАЖНО

При ревизии и удалении карточек **редактировать `src/data/words.ts` НАПРЯМУЮ**, а НЕ удалять строки из `data/raw/*.csv` с пересборкой.

Причина: `scripts/build-words.mjs` генерирует ID последовательно (`airport_001`, `airport_002`...). Если удалить строку из CSV и пересобрать, все последующие ID сдвинутся → у пользователей в IndexedDB прогресс «съедет» на соседние карточки (например, прогресс по `airport_044 layover` начнёт ссылаться на `airport_044 transit`). Прямое редактирование `words.ts` сохраняет стабильные ID.

Побочный эффект: `data/raw/*.csv` расходятся с `words.ts`. Это терпимо — `words.ts` де-факто становится source of truth. Если в будущем понадобится снова собирать из CSV, сначала надо рефакторить `build-words.mjs` на стабильные ID (например, хеш от `topicId+english+russian`).
